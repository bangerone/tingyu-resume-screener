# D6.5 — 演示加固（部署前护栏）

## Status: ✅ Done

## Goal
D7 部署到 EdgeOne Pages 后会把 URL 发给面试官，对外开放几周。不想因此被薅 LLM 账单、脏数据污染 DB、飞书群被无关推送骚扰、或 URL 被二次传播。

在不改产品行为的前提下，加 3 层护栏：
1. 访问码门禁（防止链接被爬虫 / 二次分享）
2. HR 只读模式（保留 UI 展示，但任何写操作被拒）
3. 业务维度限额（防止 LLM 账单失控 + 防止单人重复投递 / 炸飞书）

所有护栏通过 env 开关；env 未设置 → 完全跳过，本地 dev 无感。

- D6 交付物（commit 7892c18）
- 现有 auth 链路（`src/middleware.ts` / `src/lib/auth/*`）

## Deliverables（5 个措施）

### ① HR 只读模式 — `DEMO_READONLY_HR=true`
- `src/lib/auth/demo-guard.ts`（新）
  - `isReadOnlyHr(): boolean`
  - `readOnlyHrResponse(): NextResponse` → 403 `{ error: "演示模式：只读访问" }`
- 接入 4 个 HR 写入型 API（auth 校验通过后立即判断）：
  - `POST /api/jobs` — `src/app/api/jobs/route.ts`
  - `PATCH/DELETE /api/jobs/[id]` — `src/app/api/jobs/[id]/route.ts`
  - `POST /api/feishu/test` — `src/app/api/feishu/test/route.ts`
  - `POST /api/score/[id]` — `src/app/api/score/[id]/route.ts`
    - 特殊：仅在 **非内部调用** 时命中（保证 `/api/applications` 的 fire-and-forget 评分链路不断）
- `POST /api/jobs/generate-criteria` **不拦**（LLM 是产品卖点；它本身不写 DB，靠措施③防账单）
- UI：`src/app/admin/layout.tsx` 顶部黄色 banner「🔒 演示模式 · HR 只读访问，写操作会被拒绝」
  - `isReadOnlyHr()` 在 server component 里直接调；按钮保持原状（不 disable），让面试官看到完整 UI，点了由前端 toast 报错

### ② 候选人上传次数限制 — `DEMO_MAX_APPLICATIONS_PER_CANDIDATE=5`
- 两处入口都检查（防绕过）：
  - `POST /api/resume/parse`
  - `POST /api/applications`
- 逻辑：`SELECT COUNT(*) FROM applications WHERE candidate_id=?` >= cap → 429 `演示限额：每账号最多可投递 N 次`
- env 不设置或为 0 → 跳过

### ③ JD→criteria 每日全局限额（固定 10 次 / 24h）
- `src/lib/rate-limit.ts`（新）
  - 进程内存 `Map<string, Bucket>`
  - `bumpAndCheck(key, limit, windowMs): { ok, remaining }`（fixed-window）
- `/api/jobs/generate-criteria` 入口：`bumpAndCheck("generate-criteria|daily", 10, 24*3600*1000)` → 超限 429 `演示限额：AI 生成已达今日上限`
- 进程重启会清零（demo 场景不引入 Redis）

### ⑤ 访问码门禁 — `DEMO_ACCESS_CODE=<code>`
- `src/middleware.ts` 扩展：
  - matcher 改为 `["/((?!_next/static|_next/image|favicon.ico).*)"]`（跑在所有路径，除静态资源）
  - 内部放行：`/api/**`、`/_next/**`、`/favicon.ico`（API 各自守鉴权；候选人登录链路不能被门禁挡）
  - `/welcome?k=<code>` 命中 env → `Set-Cookie: demo-access=1; HttpOnly; SameSite=Lax; Max-Age=2592000` + 302 到 `next`
  - 其他路径缺 `demo-access` cookie → 302 到 `/welcome?next=<pathname+search>`
  - `safeNext()` 只接受以 `/` 开头、非 `//`、不含 `://` 的 next，否则降级为 `/`（防 open redirect）
  - env 未设置 → 完全跳过
- `/admin/*` 守卫逻辑保留在访问码门禁之后执行
- `src/app/welcome/page.tsx`（新）：
  - 简洁落地页（庭宇 logo + 「请使用面试官发送的链接进入演示环境」）
  - 内置手动输码表单（`<form method="get" action="/welcome">`）
  - 若 URL 带 `?k=xxx` 但 middleware 没放行（说明 k 错误）→ 红色错误条
- HR 登录页 `/admin/login` 也被门禁挡住（正确行为：面试官先过门禁，再看到 HR 入口）
