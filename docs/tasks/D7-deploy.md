# D7 — 部署 + 录屏 + 提交材料

## Status: ⏳ Pending

## Goal
交付：在线 demo 链接 + 设计文档 + 演示视频。

## Inputs
- 所有前置任务必须 ✅
- Vercel 账号 / 飞书测试群 / Supabase 项目都齐全

## Deliverables
- 部署到 Vercel，拿到 https://xxx.vercel.app
- 在 Vercel 配齐所有环境变量
- 录制 3-5 分钟演示视频（mp4 / mov）
- `docs/submission.md` — 1-2 页设计要点文档（提交给庭宇）
- `README.md` 更新：在线地址 + HR 测试密码 + 截图

## Steps

### 1. 部署
1. push 代码到 GitHub repo
2. Vercel import → 选 Next.js
3. Environment Variables 复制 `.env.local` 全部内容（注意 `APP_BASE_URL` 改成 vercel 域名）
4. Deploy
5. 部署后用真实流程跑一遍：建岗 → 投递 → 看分 → 收到飞书消息

### 2. 测试数据准备
- 创建 3 个 sample 岗位（前端、产品、后端）
- 用 3-5 份真实/拟合简历投递，让 dashboard 看起来不空

### 3. 录屏（OBS / QuickTime）
脚本：

| 时间 | 内容 |
|---|---|
| 0:00-0:20 | 标题画面 + 一句话定位（"AI 简历筛选 SaaS"） |
| 0:20-0:50 | 候选人视角：进 /jobs 看列表 → 进详情 → 投递表单 → 提交后看到「已收到」 |
| 0:50-1:30 | HR 视角：登录 → 列表页看到刚投递的候选人在评分中 → 几秒后变 scored |
| 1:30-2:30 | 进详情：展示评分维度 / 硬性卡 / 雷达图 / 简历预览 |
| 2:30-3:00 | 切回飞书群，展示自动收到的高匹配候选人卡片 |
| 3:00-3:30 | 回 HR 后台展示 JD 一键生成 criteria（如做了） |
| 3:30-4:30 | 技术架构口播 + 关键设计决策（评分透明化 / criteria 结构化 / 简历去重） |
| 4:30-5:00 | 项目地址 + 联系方式 + 完 |

### 4. 提交文档（docs/submission.md）

结构（**这才是面试官认真读的**）：
1. **产品定位** — 一句话
2. **用户角色 & 核心闭环图**
3. **关键设计决策表**（直接搬之前讨论的那张）
4. **数据模型**（4 张表的关系图 + 1 句话作用）
5. **AI 评分设计**（两次调用 / 维度 / 透明化策略）
6. **取舍与边界**（砍了什么、为什么）
7. **如不限时还想加什么**（体现产品 sense）

格式：可以写成 Markdown → 导出 PDF；或直接 Notion 公开页面。

- [ ] submission 文档 ≤ 2 页 A4，重点突出
- [ ] README 包含：在线地址 / HR 测试密码 / 截图 / 提交材料链接
- [ ] git 历史干净（每天一个 commit 或合并成有意义的 milestone commit）
