> 这是一份 ~10 分钟就能看完整个产品的导览。所有功能都真实接通：真 LLM 解析简历、真 LLM 评分、真飞书群推送。

## 一、如何查看演示

本项目**没有提供公网 demo 链接**（国内可用且支持 Node runtime + pdf-parse 的免费托管平台较少，权衡后选择以视频形式提交）。推荐按下面顺序查看：

1. **先看演示视频** `docs/demo.mp4` · 5 分钟走完候选人投递 → HR 后台 → 飞书推送全流程
2. **看 PPT 设计文档** `docs/presentation.pptx` · 16 页，产品定位 + 关键设计决策 + 技术栈
3. **看代码仓库** https://github.com/bangerone/tingyu-resume-screener · 按 `docs/tasks/D0-D7` 任务卡顺序可还原 1 周 vibe-coding 过程
4. **如需本地跑起来**，看 [README.md](../README.md) 的「本地开发」一节；env 齐全后 `npm run dev` 即可访问 `http://localhost:3000`

## 二、产品卖点（30 秒电梯版）

HR 建一个岗位时**用自然语言写 JD**，系统调 LLM 自动生成结构化筛选标准（硬性/软性/技能/加分）。候选人上传简历后，系统再调一次 LLM 解析简历**自动填写申请表**——候选人不用重复敲键盘。后台异步评分，高分自动推飞书群通知 HR，附 5 维度雷达图。

## 四、技术栈速览（国内栈）

| 层 | 选型 |
|---|---|
| Framework | Next.js 14 App Router + TypeScript |
| DB | TiDB Cloud Serverless（MySQL 协议） |
| ORM | Drizzle |
| 对象存储 | 腾讯云 COS（STS 临时凭证前端直传） |
| LLM | 智谱 GLM（OpenAI 兼容，国内走得通） |
| 邮件 | Resend（dev fallback 到控制台） |
| 部署 | 本地运行演示（原计划 EdgeOne Pages；因 pdf-parse/mammoth 依赖 Node runtime，Edge runtime 平台不兼容） |
| 鉴权 | HR 共享密码 + JWT；候选人邮箱验证码 + JWT |
| 推送 | 飞书自定义机器人 webhook |

## 五、几个可能被问到的点

- **为什么 LLM 调用走后端不走前端？** 保护 API key，同时避免 RSC hydration 问题，所有 AI prompt / zod 校验 / 重试 1 次都在 server route handler。
- **防 prompt injection / 防简历太大？** 简历原文截断到 12000 字符；LLM 输出全部过 zod schema 校验，不合法就重试一次，再失败标记 `status=failed`（不会挂掉整个投递链路）。
- **飞书签名？** 用 FEISHU_WEBHOOK_SECRET + timestamp + HMAC-SHA256，实时生成。
- **为什么候选人侧不展示评分？** 产品设计：候选人只看到「评估中 / 已收到」；评分只对 HR 可见，防止候选人反推算法、防止投诉评分偏见。
- **演示加固做了什么？** HR 只读模式 + 候选人投递数上限 + LLM 生成每日上限 + 访问码门禁。详见 `docs/tasks/D6.5-demo-hardening.md`。

## 六、如果遇到问题

- 页面卡在"评估中"？—— 后台 LLM 同步大约 10-30 秒，刷新一下即可

- 演示限额触发（429）？—— 说明你玩得很起劲 😄，告诉作者即可调整
